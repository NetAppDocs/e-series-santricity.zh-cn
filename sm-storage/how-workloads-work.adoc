= SANtricity 软件中的工作负载工作原理
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
创建卷时、您可以选择一个工作负载来为特定应用程序自定义存储阵列配置。

工作负载是指支持应用程序的存储对象。您可以为每个应用程序定义一个或多个工作负载或实例。



== 创建工作负载

您必须手动指定何时要创建未与特定应用程序关联的工作负载。您还必须使用添加卷页面手动指定卷配置。



== 应用程序和工作负载视图

要查看应用程序和工作负载、请启动SANtricity System Manager。在此界面中、您可以通过多种不同方式查看与应用程序特定工作负载关联的信息：

* 您可以选择 Volumes 页面中的 *Applications and workloads* 选项卡，以查看按工作负载和工作负载关联的应用程序类型分组的存储阵列的卷。
* 您可以选择性能页面中的*应用程序和工作负载*选项卡，以查看逻辑对象的性能指标（延迟、IOPS 和 MB）。对象按应用程序和相关工作负载分组。通过定期收集此性能数据，您可以建立基线测量并分析趋势，这有助于调查与 I/O 性能相关的问题。
